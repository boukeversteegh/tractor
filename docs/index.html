Quick Start

1. See the tree ‚Äî run without a query to inspect the XML structure:

tractor Program.cs

2. Query it ‚Äî use XPath to find what you need:

tractor Program.cs -x "method/name"          # all method names
tractor Program.cs -x "method[public]"        # public methods only
tractor Program.cs -x "method[async]"         # async methods
tractor Program.cs -x "count(class)"          # count classes

3. Use in CI ‚Äî fail the build on bad patterns:

tractor src/**/*.cs -x "method[async][type='void']" --expect none \
    --format gcc --message "async void is dangerous"

Why XPath?

Other code search tools use custom query languages. When your query doesn't match, you're left guessing why. With tractor, you can see the tree you're querying.

The workflow: tractor file.cs ‚Üí see the XML ‚Üí write your XPath ‚Üí test in any XPath tool ‚Üí -x "..." ‚Üí done.

Transparent

Run tractor file.cs to see exactly what you're querying. No hidden structure.

Debuggable

Copy XML into browser DevTools or any XPath tester. Refine with instant feedback.

Standard

XPath is a W3C standard. Millions know it. Stack Overflow has 50k+ answers.

Readable

//method[public][async] reads like English. No DSL to memorize.

Refactoring

Reference

Element Names

The semantic tree uses intuitive, lowercase names:

# Structure
class, struct, interface, method, property, field, param, block, name, type

# Modifiers (empty elements)
public, private, static, async, readonly, abstract, virtual, override

# Statements
if, else, for, foreach, while, try, catch, throw, return, await

Output Formats

-f lines    # Source code snippets (default)
-f xml      # XML fragments
-f json     # JSON array with file, line, value
-f gcc      # GCC-style for IDE integration
-f count    # Just the count

All Options

tractor <files...> [options]

-x, --xpath <expr>    XPath 2.0 query (omit to see full tree)
-e, --expect <val>    Expected: none, some, or a number
-f, --format <fmt>    Output format (see above)
-m, --message <msg>   Custom message ({value}, {line}, {xpath})
--color <mode>        auto | always | never
-h, --help            Show help
