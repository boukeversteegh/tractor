What is it?

tractor is a command-line tool that converts source code into an XML AST (Abstract Syntax Tree) and lets you query it using XPath 2.0 expressions. Think of it as grep, but for code structure instead of text.

Structural Search

Find code patterns by structure, not just text. Search for methods, classes, if statements, and more.

XPath 2.0

Use the full power of XPath 2.0 with predicates, functions, and axes for complex queries.

CI/CD Ready

Use --expect and --format gcc to enforce code patterns in your pipeline.

Colorized Output

Beautiful syntax-highlighted XML output in your terminal with --color support.

Why XPath?

Other code search tools introduce custom query languages that are powerful but opaque. When your query doesn't match, you're left guessing why.

See What You're Querying

Run tractor file.cs without any query to see the full XML AST. No hidden structure‚Äîjust look at the output and write your XPath.

Debug With Any XPath Tool

Copy the XML into browser DevTools, an online XPath tester, or VS Code. Test and refine your query with instant feedback.

No New Syntax to Learn

XPath is a W3C standard from 1999. Millions of developers already know it. Stack Overflow has 50k+ XPath questions with answers.

Predicates That Make Sense

[@modifiers='public'] reads like English. No magic symbols or pattern-matching DSLs to memorize.

The workflow: Run tractor file.cs ‚Üí see the XML ‚Üí write your XPath ‚Üí test in any XPath tool ‚Üí run with -x "..." ‚Üí done.

Quick Examples

View the full XML AST of a file:

tractor Program.cs

Find all method names:

tractor Program.cs -x "//Method/@name"

Find all public methods:

tractor Program.cs -x "//Method[@modifiers='public']/@name"

Find if statements inside methods:

tractor Program.cs -x "//Method//if"

Count the number of classes:

tractor Program.cs -x "count(//class)"

CI check - fail if any method has more than 50 lines:

tractor src/**/*.cs -x "//Method[@endLine - @startLine > 50]" \
    --expect none --format gcc --message "Method too long"

Element Names

The XML uses intuitive element names:

Keywords (lowercase)

if, else, for, foreach, while, class, struct, interface, namespace, using, try, catch, throw, return, await, new, ...

Structural (PascalCase)

Method, Property, Field, Constructor, Param, Block, Call, Arg, Lambda, Assign, ...

Output Formats

Use -f or --format to change the output:

  • lines - Source code snippets (default)
  • xml - XML fragments for each match
  • json - JSON array with file, line, column, value
  • gcc - GCC-style error format for IDE integration
  • count - Just the number of matches

All Options

Usage: tractor <files...> [options]
       tractor --stdin [options]

Options:
  -x, --xpath <expr>     XPath 2.0 query (without: outputs full XML AST)
  -e, --expect <value>   Expected result: none, some, or a number
  -f, --format <fmt>     Output format: lines, gcc, json, count, xml
  -m, --message <msg>    Custom message (supports {value}, {line}, {xpath})
  --keep-locations       Include start/end line+col attributes in XML output
  --color <mode>         Color output: auto (default), always, never
  --no-color             Disable colored output
  -v, --verbose          Show parsed arguments (debug shell escaping)
  -h, --help             Show help
